## Tooltip

The Tooltip component is a fundamental UI element that displays additional information when users interact with elements. It's perfect for providing context, explanations, or supplementary details without cluttering the interface.

## Basic Usage

<OnHoverExample />

## Features

- **Flexible Positioning**: Position tooltips on any side (top, right, bottom, left)
- **Automatic Positioning**: Smart positioning that adjusts based on available space
- **Accessible**: Built on Radix UI primitives with full keyboard and screen reader support
- **Customizable Styling**: Easy to customize appearance with CSS classes
- **Animation**: Smooth enter/exit animations
- **Delay Control**: Configurable delay before showing/hiding tooltips

## Components

The Tooltip system consists of several components that work together:

### TooltipProvider
Wraps your application to provide tooltip context and configuration.

### Tooltip
The root component that manages the tooltip state.

### TooltipTrigger
The element that triggers the tooltip when hovered or focused.

### TooltipContent
The actual tooltip content that appears.

## Props

### TooltipProvider Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| delayDuration | number | 700 | Delay before showing the tooltip in milliseconds |
| skipDelayDuration | number | 300 | Delay before hiding the tooltip in milliseconds |
| children | ReactNode | - | The content to render |

### Tooltip Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| defaultOpen | boolean | false | Whether the tooltip is open by default |
| open | boolean | - | Controlled open state |
| onOpenChange | (open: boolean) => void | - | Callback when open state changes |
| delayDuration | number | - | Override provider delay duration |
| disableHoverableContent | boolean | false | Whether to disable hoverable content |

### TooltipTrigger Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| asChild | boolean | false | Whether to render as a child element |
| children | ReactNode | - | The trigger element |
| className | string | - | Additional CSS classes |

### TooltipContent Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| side | "top" \| "right" \| "bottom" \| "left" | "top" | The side where the tooltip appears |
| align | "start" \| "center" \| "end" | "center" | The alignment of the tooltip |
| sideOffset | number | 4 | The offset from the side in pixels |
| alignOffset | number | 0 | The offset from the alignment in pixels |
| avoidCollisions | boolean | true | Whether to avoid collisions with viewport edges |
| collisionBoundary | Boundary | [] | The boundary for collision detection |
| collisionPadding | number \| Padding | 0 | The padding for collision detection |
| arrowPadding | number | 0 | The padding for arrow collision detection |
| sticky | "partial" \| "always" | "partial" | The sticky behavior |
| hideWhenDetached | boolean | false | Whether to hide when detached |
| className | string | - | Additional CSS classes |
| children | ReactNode | - | The tooltip content |

## Examples

### Basic Tooltip
```tsx
<TooltipProvider>
  <Tooltip>
    <TooltipTrigger>
      <button>Hover me</button>
    </TooltipTrigger>
    <TooltipContent>
      This is a basic tooltip
    </TooltipContent>
  </Tooltip>
</TooltipProvider>
```

### Custom Positioning
```tsx
<TooltipProvider>
  <Tooltip>
    <TooltipTrigger>
      <button>Hover me</button>
    </TooltipTrigger>
    <TooltipContent side="right" align="start" sideOffset={8}>
      Tooltip on the right
    </TooltipContent>
  </Tooltip>
</TooltipProvider>
```

### Custom Styling
```tsx
<TooltipProvider>
  <Tooltip>
    <TooltipTrigger>
      <button>Hover me</button>
    </TooltipTrigger>
    <TooltipContent className="bg-blue-500 text-white border-blue-600">
      Custom styled tooltip
    </TooltipContent>
  </Tooltip>
</TooltipProvider>
```

### Controlled State
```tsx
const [open, setOpen] = useState(false);

<TooltipProvider>
  <Tooltip open={open} onOpenChange={setOpen}>
    <TooltipTrigger>
      <button>Click me</button>
    </TooltipTrigger>
    <TooltipContent>
      Controlled tooltip
    </TooltipContent>
  </Tooltip>
</TooltipProvider>
```

## Accessibility

The Tooltip component follows accessibility best practices:

- **Keyboard Navigation**: Tooltips can be triggered and dismissed using keyboard
- **Screen Reader Support**: Proper ARIA attributes and announcements
- **Focus Management**: Automatic focus handling for keyboard users
- **High Contrast**: Compatible with high contrast mode
- **Reduced Motion**: Respects user's motion preferences

## Best Practices

1. **Keep content concise**: Tooltips should provide quick, helpful information
2. **Use appropriate timing**: Default delays work well for most cases
3. **Position thoughtfully**: Consider the user's reading flow and available space
4. **Test with keyboard**: Ensure tooltips work for keyboard-only users
5. **Avoid overuse**: Don't use tooltips for essential information
