## Overview

The Table component is a powerful data table built with **@tanstack/react-table** and **@tanstack/react-virtual**. It provides enterprise-grade features including virtual scrolling for performance, sorting, filtering, column resizing, row selection, and data export capabilities.

## Usage

```tsx
import { Table, type ColumnDef } from "@unkey/ui";

interface User {
  id: string;
  name: string;
  email: string;
  role: string;
}

const columns: ColumnDef<User>[] = [
  { accessorKey: "name", header: "Name" },
  { accessorKey: "email", header: "Email" },
  { accessorKey: "role", header: "Role" },
];

export default function MyComponent() {
  const [users, setUsers] = useState<User[]>([]);

  return (
    <Table
      data={users}
      columns={columns}
      enableSorting
      enableVirtualization
    />
  );
}
```

## Features

### Core Features
- **Virtual Scrolling**: Handles 10,000+ rows with excellent performance using `@tanstack/react-virtual`
- **State Management**: Full table state management with `@tanstack/react-table`
- **TypeScript First**: Complete type safety with generic data types
- **Responsive**: Works on all screen sizes with proper overflow handling

### Data Operations
- **Sorting**: Single and multi-column sorting (client & server-side)
- **Filtering**: Column filters and global search
- **Pagination**: Client-side, server-side, or infinite scroll modes
- **Export**: CSV and JSON export with customizable options

### Column Features
- **Resizing**: Drag to resize columns with min/max constraints
- **Reordering**: Drag and drop to reorder columns (coming soon)
- **Visibility**: Toggle column visibility with built-in UI
- **Custom Rendering**: Full control over cell and header rendering

### Interaction
- **Row Selection**: Single or multi-row selection
- **Keyboard Navigation**: Arrow keys and Vim bindings (j/k)
- **Row Actions**: Click handlers for row interactions
- **Loading States**: Skeleton loaders and custom loading states

### Persistence
- **LocalStorage**: Save column sizes, order, and visibility
- **Automatic**: Debounced saves to prevent performance issues
- **Per-Table**: Unique persistence keys for multiple tables

## Column Definition

Columns use the standard `@tanstack/react-table` `ColumnDef` type:

```tsx
const columns: ColumnDef<User>[] = [
  {
    accessorKey: "name",
    header: "Name",
    size: 200,
    minSize: 100,
    maxSize: 400,
    enableSorting: true,
    enableResizing: true,
    cell: ({ row }) => (
      <div className="font-medium">{row.getValue("name")}</div>
    ),
  },
];
```

### Column Properties

- **accessorKey**: Key to access data in row object
- **header**: Column header (string or component)
- **cell**: Custom cell renderer
- **size**: Default column width
- **minSize/maxSize**: Min/max width for resizing
- **enableSorting**: Allow sorting this column
- **enableResizing**: Allow resizing this column
- **sortingFn**: Custom sorting function

## Performance

### Virtual Scrolling
The table uses virtual scrolling to render only visible rows:

```tsx
<Table
  data={largeDataset}
  columns={columns}
  enableVirtualization={true}
  estimateSize={52}  // Row height in pixels
  overscan={5}       // Extra rows to render
/>
```

### Best Practices
- Enable virtualization for > 100 rows
- Use `useMemo` for column definitions
- Use `getRowId` for stable row keys
- Enable server-side operations for very large datasets

## Server-Side Operations

For large datasets, use server-side operations:

```tsx
const [sorting, setSorting] = useState<SortingState>([]);
const [data, setData] = useState([]);

// Fetch data when sorting changes
useEffect(() => {
  fetchData({ sorting }).then(setData);
}, [sorting]);

<Table
  data={data}
  columns={columns}
  enableSorting={true}
  manualSorting={true}
  sorting={sorting}
  onSortingChange={setSorting}
/>
```

## Migration from Old Tables

If migrating from the previous VirtualTable implementation:

1. **Column definitions**: Use TanStack Table's `ColumnDef` instead of custom column format
2. **Sorting**: Built-in instead of manual implementation
3. **State management**: Automatic instead of manual hooks
4. **Filtering**: Built-in instead of custom filters
5. **Export**: Built-in instead of custom implementation

Example migration:

```tsx
// Old VirtualTable
<VirtualTable
  data={data}
  columns={[
    { key: 'name', render: (item) => item.name }
  ]}
/>

// New Table
<Table
  data={data}
  columns={[
    { accessorKey: 'name', header: 'Name' }
  ]}
/>
```

## Architecture

The table component integrates two powerful libraries:

- **@tanstack/react-table**: Handles table state, sorting, filtering, pagination
- **@tanstack/react-virtual**: Provides virtual scrolling for performance

This combination provides:
- Full feature set from TanStack Table
- Excellent performance with virtual scrolling
- Standard API that many developers know
- Active community and extensive documentation
